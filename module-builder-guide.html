🗺 Overview

What is Module Builder?

Module Builder is a browser-based tool for TVET curriculum developers to design competency-based training modules and export them as formatted Word (.docx) documents — with no installation, no internet connection required after loading.

🏗
Module-based structureOrganize content by Modules → Learning Outcomes → Info & Activity Sheets
📄
Export to Word (.docx)Full module exported as formatted document with all sheets, tables & images
💾
Save & Load (JSON)Save your entire work as a JSON file and restore it anytime
🔁
AutoSaveBackground autosave to browser localStorage every ~1 second of inactivity
🏷
11 Instructional MarksTip, Warning, Note, Example, and more — for rich pedagogical annotation
📋
Inline TablesAdd custom tables inside content sections for structured information
🖼️
Multiple imagesAdd multiple images per content section or activity step with delete controls
🔢
Auto-numberingSheets auto-numbered by LO index (e.g. 1-1, 1-2, 2-1)
🔧 Toolbar

Top Toolbar

The fixed toolbar at the top gives you instant access to the core actions from any tab.

💾 Save Work 📂 Load Work 📄 Export to Word 🗑 Clear All
ButtonWhat it does
💾 Save WorkDownloads a .json backup file named ModuleTitle_YYYY-MM-DD.json. Contains all modules, sheets, images, and assessment data.
📂 Load WorkOpens a .json file previously saved with Save Work, restoring all data.
📄 Export to WordGenerates a complete .docx file named ModuleTitle_YYYY-MM-DD.docx covering all LOs and all sheets in order.
🗑 Clear AllPermanently clears all entered data across every tab. Shows a confirmation dialog first.

1

2

3

Learning Outcome ManagementAdd, rename, and reorder LOs

1

📋 Introduction

Introduction Pages Tab

Add a development team table and introductory text that appear before the module content.

1

Click ➕ Add Team Member to add rows to the Work Team table. Each row has: Name, Task/Role, and Work Location.

2

Use the Additional Introduction Details textarea for acknowledgments, a foreword, or any introductory text. This starts on a new page if it contains content.

ℹ️
The Module Overview (list of all LOs and their Performance Criteria) is generated automatically in the export — no manual input needed.
📄 Information Sheet

Information Sheet Tab

Create content-rich information sheets for each Learning Outcome. Each LO can have multiple Information Sheets, auto-numbered (e.g. 1-1, 1-2, 2-1).

📋

Sheet NavigationAdd, browse and remove sheets per LO

When you open the Information Sheet tab, the first sheet is created automatically. The Sheet N of N counter shows your position.

◀ Prev Sheet 1 of 2 Next ▶ ➕ Add Info Sheet 🗑 Remove Sheet
✏️

Content SectionsAdd multiple content blocks per sheet

1

Enter text in the Content textarea. Use the ➕ Add Content button to add more blocks below.

2

Each content block can have: 🖼️ Images (multiple), 📋 Tables (inline), and 🏷 Instructional Marks.

3

Use ✕ to clear a block's content, or 🗑 to delete the entire block.

💡
In Word export, text appears on the left column (8cm) and the first image on the right (8cm). Additional images export as extra rows below.
📋

Inline TablesAdd structured data tables inside content

1

Click 📋 Add Table to insert an editable table (2×2 by default) below a content block.

2

Use ➕ Row ➕ Col to expand, or 🗑 Row 🗑 Col (right side) to shrink.

3

Table content is saved in JSON and exported to Word as a full-width 16cm table.

🔍

Self-check & Answers-KeyAuto-numbered from the sheet number

Both sections appear on separate pages after the information sheet in the exported document. The number field auto-fills from the sheet number.

No: 🔢 Number • Bullet ✖ Clear 🗑

✖ Clear removes numbering/bullet formatting. 🗑 clears both the number and all content.

🔗

QR Code & Video LinkOptional resource link at bottom of sheet

Upload a QR code image and enter a video URL and subject line. These appear as a small table at the bottom of the information sheet in the Word export.

2

Performance Criteria Check ListAlways visible — auto-numbered from sheet number

✅ Assessment Unit

Assessment Unit Tab

Create Portfolio of Evidence assessment forms, one per Learning Outcome. Forms auto-populate when you open this tab.

1

Open the Assessment Unit tab — forms are auto-created for every existing LO with 5 empty rows each.

2

Each form contains: Portfolio of Evidence table (Assessment Criteria, Activities, Outcomes, Verification, Date), Result checkboxes (Competent / Not Yet Competent), and a Signatures section (printed manually).

3

Add rows with ➕ Add Row, clear with 🗑️ Clear Rows, or delete the whole form with ❌ Delete Form.

4

Only forms with at least one filled row are exported to Word — blank forms are skipped.

⚠️
Use ➕ Add Assessment Form to manually add a form for any new LO added after the tab was first opened.

Add bibliography entries. Exported as the last content section before the back cover.

2

Use ✏️ Rename to change the section heading (default: "References").

Empty entries are skipped in the Word export.

Add contextual callout boxes to any Content block (Info Sheet) or Activity Step. Click 🏷 Add Mark to open the dropdown.

Warns learners about important points that need care
🔍 Review
Prompts revisiting key information before proceeding
❓ Question
A reflective question to engage critical thinking
💭 Reflect
Encourages deeper thinking or self-assessment
📝 Note
A supplementary note or clarification
💡 Tip
A practical hint or shortcut for the learner
❗ Important
Critical information that must not be missed
🔔 Remember
Key facts or steps to memorize
⛔ Warning
Safety or procedural warnings
📌 Example
A worked example or sample

Best practice or recommended approach
💡
Marks are exported to Word as 2-row colored tables (header row + content row) at full 16cm width. They are saved in JSON and restored on load.
💾 AutoSave

AutoSave & Backup Reminder

Module Builder includes a background autosave system to protect your work from accidental data loss.

🔄
AutoSaveSaves to browser localStorage every ~900ms after you stop typing. A small green dot (●) flashes bottom-right on each save.
✅
Session RestoreOn page load, if autosave data is found, it restores automatically and shows "Previous session restored" for 6 seconds.
⏰
Backup ReminderEvery 8 minutes, a subtle toast appears: "Don't forget to save your work" with a [Save Now] button.
📂
Manual Save = Best PracticeAutoSave is a safety net, not a replacement. Use 💾 Save Work regularly for a portable JSON backup.
⚠️
AutoSave data is stored in browser localStorage. Clearing browser data, using a different browser, or switching devices will lose the autosave. Always keep a downloaded JSON backup.
💡 Pro Tips

Tips & Best Practices

💾
Save oftenClick Save Work after completing each sheet. The file is named with the module title and date for easy versioning.
🔢
Let auto-numbering workSheet numbers like 1-1, 2-3 are assigned automatically. Only edit the number if you need a custom value.
📋
Use inline tables for specsAdd a 📋 Table to present technical specifications, dimensions, or comparisons inside a content block.
🏷
Use marks for pedagogyAdd a ✅ Good Practice or ⚠️ Attention mark after key steps to guide the learner's attention.
📦
One module per fileFor large programmes with multiple modules, keep separate JSON files per module for easier management.
🖼️
Image resolutionUse images at least 800px wide for best Word export quality. PNG or JPG both work.
🔄
Load = full restoreLoading a JSON file restores every module, LO, sheet, image, mark, table, and assessment form exactly as saved.
📱
Works on mobileThe interface is responsive. You can review content on a phone, though data entry is most comfortable on desktop.
